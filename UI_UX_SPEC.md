# UI/UX Specification: The Scrolling Newspaper (Updated)

## Navigation Paradigm
The core interaction model is a dual-axis "Intelligence Map."

### 1. The Horizontal Axis (The Timeline)
The user moves horizontally to navigate through time.
- **Left:** The Past (Historical context, "The Roots").
- **Center:** The Present (Modern manifestations, "The Echoes").
- **Right:** The Future (Prophetic anticipation, "The Horizon").

### 2. The Vertical Axis (The Content)
Within any point in the horizontal timeline, the user scrolls vertically to consume the specific "Digest" entries.

## The Oracle: Search as a Lens
A central, high-performance search interface that filters the entire digest in real-time.

### Interaction Model:
- **Natural Language Intent:** Users can search for themes (e.g., "signs of the end") rather than just keywords.
- **Dynamic Highlighting:** Instead of a separate results page, the search "filters" the newspaper. Irrelevant articles fade; relevant ones are highlighted and brought into focus.
- **Timeline Navigation:** Searching for a "Past" event automatically glides the horizontal timeline to that era. Searching for a "Future" prophecy glides it to the "Horizon."
- **Visual Feedback:** A "Command Palette" style overlay provides instant feedback as the user types.

## Visual & Typographic Style
- **Style:** Traditional newspaper layout, beautiful typography (Serif fonts for body, high-quality headers).
- **Layout:** Multi-column flow, dense but readable.
- **Visual Transitions:** Subtle color/style shifts as the user moves from "Roots" (Archival) to "Echoes" (Modern/High-Contrast) to "Horizon" (Minimalist/Impactful).
